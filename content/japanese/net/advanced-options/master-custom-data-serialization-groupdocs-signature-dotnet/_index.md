### シリアル化属性を持つカスタムデータクラスの定義
GroupDocs.Signatureでは、属性を使用してカスタムデータシリアル化ルールを定義できます。ドキュメント署名用のクラスを作成する方法は次のとおりです。
#### 概要
カスタムシリアル化により、データは保存または送信される前に特定の要件に従ってフォーマットされます。このセクションでは、カスタムシリアル化クラスの作成と設定方法を説明します。
#### ステップバイステップの実装
**1. データクラスを作成する**
まず、属性を使用してシリアル化形式を指定し、ID、作成者、日付のプロパティを持つクラスを定義します。
```csharp
using System;
using GroupDocs.Signature.Domain.Extensions;

public class DocumentSignatureData
{
    // シリアル化形式を指定するには、Format 属性を使用します。
    [Format("SignID")]
    public string ID { get; set; }

    [Format("SAuth")]
    public string Author { get; set; }

    [Format("SDate")]
    public DateTime Date { get; set; }
}
```
**2. パラメータと属性を説明する**
- `Format("SignID")`この属性は、ID プロパティのシリアル化された出力にカスタム名 (「SignID」) を割り当てます。
- **目的**これらの属性により、データ クラスがシリアル化されるときに、各プロパティが指定された形式にマップされ、他のシステムとの互換性が向上します。
#### 主要な設定オプション
