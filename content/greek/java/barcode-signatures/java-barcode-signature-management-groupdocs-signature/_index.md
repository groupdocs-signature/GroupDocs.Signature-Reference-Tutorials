### Αναζήτηση Υπογραφών Barcode

**Anchor ορισμού:** Η κλάση `BarcodeSearchOptions` ρυθμίζει τα κριτήρια που χρησιμοποιεί η **java electronic signature library** για τον εντοπισμό υπογραφών barcode μέσα σε ένα έγγραφο.

#### Άμεση απάντηση
Καλέστε τη μέθοδο `search()` στο αντικείμενο `Signature`, περνώντας ένα αντικείμενο `BarcodeSearchOptions`. Η μέθοδος επιστρέφει λίστα αντικειμένων `BarcodeSignature` που ταιριάζουν με τα κριτήρια που ορίσατε, επιτρέποντάς σας να ελέγξετε τον τύπο, το περιεχόμενο και τη θέση κάθε barcode.

#### Βήμα 2: Ρύθμιση Επιλογών Αναζήτησης

``` 
```java
import com.groupdocs.signature.Signature;
import com.groupdocs.signature.domain.signatures.BarcodeSignature;
import com.groupdocs.signature.options.search.BarcodeSearchOptions;

public class SearchBarcodeSignatures {
    public static void run() throws Exception {
        String filePath = "YOUR_DOCUMENT_DIRECTORY/sample.pdf";
        
        final Signature signature = new Signature(filePath);
        
        // Create search options for barcode signatures
        BarcodeSearchOptions options = new BarcodeSearchOptions();
        
        // Search for barcode signatures in the document
        List<BarcodeSignature> signatures = signature.search(BarcodeSignature.class, options);
        if (!signatures.isEmpty()) {
            // Access found barcode signatures from the 'signatures' list.
        }
    }
}
```
```

**Ανάλυση:** Η κλάση `BarcodeSearchOptions` σας επιτρέπει να προσαρμόσετε την αναζήτηση. Από προεπιλογή, ψάχνει σε όλο το έγγραφο για όλους τους τύπους barcode, αλλά μπορείτε να την ρυθμίσετε ώστε:  

- Στοχεύει συγκεκριμένες μορφές barcode (Code128, QR κ.λπ.)  
- Αναζητά μόνο σε ορισμένες σελίδες  
- Φιλτράρει κατά περιεχόμενο ή μεταδεδομένα barcode  

Η μέθοδος `search()` επιστρέφει λίστα `BarcodeSignature`. Κάθε αντικείμενο περιέχει λεπτομέρειες για το barcode: θέση, περιεχόμενο, τύπο και μεταδεδομένα. Αν η λίστα είναι κενή, δεν βρέθηκαν υπογραφές barcode (που μπορεί να σημαίνει ότι το έγγραφο δεν έχει τέτοιες υπογραφές ή ότι δεν είναι ρυθμισμένες στις επιλογές σας).

**Πρακτικό παράδειγμα:** Σε σύστημα επεξεργασίας τιμολογίων, μπορείτε να αναζητήσετε υπογραφές barcode για αυτόματη εξαγωγή αριθμών τιμολογίων και κωδικών επικύρωσης, εξαλείφοντας την ανάγκη χειροκίνητης εισαγωγής δεδομένων.

### Διαγραφή Υπογραφής Barcode

**Anchor ορισμού:** Η κλάση `BarcodeSignature` αντιπροσωπεύει μια μοναδική υπογραφή ηλεκτρονικού εγγράφου βασισμένη σε barcode που εντοπίστηκε από τη **java electronic signature library**.

#### Άμεση απάντηση
Αφού εντοπίσετε το επιθυμητό `BarcodeSignature`, καλέστε τη μέθοδο `delete()` με διαδρομή εξόδου. Η μέθοδος δημιουργεί νέο αρχείο χωρίς το επιλεγμένο barcode, αφήνοντας το αρχικό έγγραφο αμετάβλητο για λόγους ελέγχου.

#### Βήμα 3: Εντοπισμός και Αφαίρεση της Υπογραφής

``` 
```java
import com.groupdocs.signature.Signature;
import com.groupdocs.signature.domain.signatures.BarcodeSignature;
import java.io.File;

public class DeleteBarcode {
    public static void run() throws Exception {
        String filePath = "YOUR_DOCUMENT_DIRECTORY/sample.pdf";
        
        final Signature signature = new Signature(filePath);
        
        BarcodeSearchOptions options = new BarcodeSearchOptions();
        
        List<BarcodeSignature> signatures = signature.search(BarcodeSignature.class, options);
        if (!signatures.isEmpty()) {
            BarcodeSignature barcodeSignature = signatures.get(0);
            
            String outputFilePath = new File("YOUR_OUTPUT_DIRECTORY", "output_sample.pdf").getPath();
            
            // Delete the first found barcode signature from the document
            boolean result = signature.delete(outputFilePath, barcodeSignature);
            if (result) {
                // Signature successfully deleted.
            } else {
                // Could not find or delete the signature.
            }
        }
    }
}
```
```

**Κατανόηση της διαδικασίας:** Αυτός ο κώδικας ακολουθεί το μοτίβο αναζήτηση‑μετά‑διαγραφή. Πρώτα βρίσκουμε όλες τις υπογραφές barcode στο έγγραφο. Στη συνέχεια παίρνουμε την πρώτη (μπορείτε να κάνετε βρόχο για όλες ή να φιλτράρετε βάσει κριτηρίων). Τέλος, καλούμε `delete()` με διαδρομή εξόδου για να αφαιρέσουμε την υπογραφή.

**Σημαντική σημείωση:** Η μέθοδος `delete()` δημιουργεί νέο έγγραφο με την υπογραφή αφαιρεμένη—δεν τροποποιεί το αρχικό αρχείο. Αυτό είναι ένα χαρακτηριστικό ασφαλείας, ώστε να διατηρείτε το αρχικό έγγραφο αν χρειαστεί. Βεβαιωθείτε ότι το `"YOUR_OUTPUT_DIRECTORY"` δείχνει σε θέση όπου έχετε δικαιώματα εγγραφής.

Η Boolean τιμή επιστροφής δείχνει αν η διαγραφή ήταν επιτυχής. Αν επιστρέψει `false`, οι πιο συνηθισμένοι λόγοι είναι:  

- Η υπογραφή δεν υπάρχει πλέον στο έγγραφο (ίσως έχει ήδη αφαιρεθεί)  
- Προβλήματα δικαιωμάτων στο φάκελο εξόδου  
- Η μορφή του εγγράφου δεν υποστηρίζει αφαίρεση υπογραφής  

**Συμβουλή:** Σε παραγωγικό κώδικα, ελέγχετε ποια υπογραφή διαγράφετε πριν καλέσετε `delete()`. Μπορείτε να ελέγξετε ιδιότητες όπως `barcodeSignature.getText()` ή `barcodeSignature.getEncodeType()` για να βεβαιωθείτε ότι αφαιρείτε τη σωστή.

## Συνηθισμένα Λάθη που Πρέπει να Αποφύγετε

Αυτά είναι τα πιο συχνά προβλήματα που αντιμετωπίζουν οι προγραμματιστές (και πώς να τα αποφύγετε):

### 1. Λανθασμένος Χειρισμός Διαδρομών Αρχείων  
**Το λάθος:** Σκληρή κωδικοποίηση διαδρομών ή παράλειψη διαχείρισης διαφορετικών διαχωριστών OS.  

**Η λύση:** Χρησιμοποιήστε `File.separator` ή προτιμήστε τις μπροστιές κάθετες (`/`) που λειτουργούν σε όλες τις πλατφόρμες. Καλύτερα, χρησιμοποιήστε `Paths.get()` από `java.nio.file` για αξιόπιστο χειρισμό:

``` 
```java
String filePath = Paths.get("YOUR_DOCUMENT_DIRECTORY", "sample.pdf").toString();
```
```

### 2. Παράλειψη Κλεισίματος Πόρων  
**Το λάθος:** Μη κλείσιμο του αντικειμένου `Signature`, οδηγώντας σε κλειδώματα αρχείων ή διαρροές μνήμης με πολλαπλά έγγραφα.  

**Η λύση:** Χρησιμοποιήστε try‑with‑resources (η κλάση `Signature` υλοποιεί `AutoCloseable`):

``` 
```java
try (Signature signature = new Signature(filePath)) {
    // Your code here
}
// Automatically closed and resources released
```
```

### 3. Υπόθεση ότι Όλα τα Barcodes Θα Βρεθούν  
**Το λάθος:** Μη έλεγχος αν η αναζήτηση επέστρεψε κενά αποτελέσματα πριν προσπελάσετε δεδομένα υπογραφής.  

**Η λύση:** Πάντα επικυρώστε τα αποτελέσματα της αναζήτησης:

``` 
```java
List<BarcodeSignature> signatures = signature.search(BarcodeSignature.class, options);
if (signatures.isEmpty()) {
    System.out.println("No barcode signatures found in the document.");
    return;
}
```
```

### 4. Παράβλεψη Συμβατότητας Μορφής Εγγράφου  
**Το λάθος:** Υπόθεση ότι όλες οι λειτουργίες δουλεύουν σε όλες τις μορφές εγγράφων.  

**Η λύση:** Ελέγξτε την τεκμηρίωση για περιορισμούς ανά μορφή. Για παράδειγμα, ορισμένες παλαιότερες μορφές μπορεί να μην υποστηρίζουν ορισμένες λειτουργίες υπογραφής.

## Οδηγός Αντιμετώπισης Προβλημάτων

Συμβαίνουν προβλήματα; Εδώ είναι λύσεις στα πιο συχνά ζητήματα:

### Πρόβλημα: Εξαίρεση "File not found"  
**Συμπτώματα:** `FileNotFoundException` κατά την αρχικοποίηση του αντικειμένου Signature.  

**Λύσεις:**  
- Ελέγξτε ξανά τη διαδρομή του αρχείου (χρησιμοποιήστε απόλυτες διαδρομές κατά τον εντοπισμό σφαλμάτων)  
- Βεβαιωθείτε ότι το αρχείο υπάρχει στη θέση αυτή  
- Ελέγξτε τα δικαιώματα—η εφαρμογή πρέπει να έχει πρόσβαση ανάγνωσης  
- Μην συγχέετε σχετικές διαδρομές έργου με απόλυτες διαδρομές συστήματος  

### Πρόβλημα: Δεν Βρέθηκαν Υπογραφές (Αν και Γνωρίζετε ότι Υπάρχουν)  
**Συμπτώματα:** Η αναζήτηση επιστρέφει κενή λίστα παρόλο που οι υπογραφές είναι ορατές στο έγγραφο.  

**Λύσεις:**  
- Οι υπογραφές μπορεί να μην είναι τύπου barcode—δοκιμάστε αναζήτηση άλλων τύπων υπογραφών  
- Οι `BarcodeSearchOptions` μπορεί να είναι πολύ περιοριστικές (δοκιμάστε προεπιλογές πρώτα)  
- Το έγγραφο μπορεί να είναι κατεστραμμένο—ανοίξτε το σε προβολέα PDF για επιβεβαίωση  
- Ορισμένα έγγραφα έχουν υπογραφές σε μη τυπικές μορφές που το GroupDocs δεν αναγνωρίζει  

### Πρόβλημα: Η Διαγραφή Αποτυγχάνει (Επιστρέφει False)  
**Συμπτώματα:** Η μέθοδος `delete()` επιστρέφει `false` και η υπογραφή παραμένει.  

**Λύσεις:**  
- Επαληθεύστε ότι έχετε δικαιώματα εγγραφής στον φάκελο εξόδου  
- Ελέγξτε ότι το αντικείμενο υπογραφής είναι ακόμα έγκυρο (τα αποτελέσματα αναζήτησης μπορούν να γίνουν παλιά)  
- Βεβαιωθείτε ότι το αρχείο εξόδου δεν είναι ανοιχτό σε άλλη εφαρμογή  
- Δοκιμάστε διαγραφή μετά από νέα αναζήτηση (αναζητήστε ξανά αμέσως πριν τη διαγραφή)  

### Πρόβλημα: OutOfMemoryError με Μεγάλα Έγγραφα  
**Συμπτώματα:** Η εφαρμογή σας καταρρέει όταν επεξεργάζεται μεγάλα PDF.  

**Λύσεις:**  
- Αυξήστε το μέγεθος heap της JVM: `-Xmx4g` (ή περισσότερο, ανάλογα με τις ανάγκες)  
- Επεξεργαστείτε έγγραφα σε παρτίδες αν διαχειρίζεστε πολλά αρχεία  
- Σκεφτείτε επεξεργασία συγκεκριμένων σελίδων αντί για ολόκληρο το έγγραφο  
- Χρησιμοποιήστε σελιδοποίηση στις επιλογές αναζήτησης για περιορισμό χρήσης μνήμης  

## Πότε να Χρησιμοποιήσετε Αυτή τη Προσέγγιση
Χρησιμοποιήστε αυτή τη μέθοδο όταν η εφαρμογή σας απαιτεί αυτοματοποιημένη διαχείριση υπογραφών barcode σε διάφορους τύπους εγγράφων. Είναι ιδιαίτερα χρήσιμη για ροές εργασίας που χρειάζονται επαλήθευση, εξαγωγή ή αφαίρεση υπογραφών σε κλίμακα, όπως επεξεργασία τιμολογίων, διαχείριση συμβάσεων ή συμμόρφωση, όπου η χειροκίνητη επιθεώρηση θα ήταν μη πρακτική.  

- ✅ Ιδανική εφαρμογή:  
  - Κατασκευή συστημάτων διαχείρισης εγγράφων όπου απαιτείται επικύρωση υπογραφών  
  - Αυτοματοποίηση ροών εργασίας συμβάσεων με επαλήθευση barcode  
  - Επεξεργασία τιμολογίων ή αποδείξεων με ενσωματωμένες υπογραφές barcode  
  - Δημιουργία αρχείων ελέγχου για υπογεγραμμένα έγγραφα  
  - Εφαρμογές που χειρίζονται πολλαπλές μορφές εγγράφων (PDF, Word, Excel)  

- ❌ Δεν είναι η κατάλληλη λύση όταν:  
  - Εργάζεστε μόνο με μία μορφή εγγράφου και έχετε ήδη μια βιβλιοθήκη γι' αυτήν  
  - Οι ανάγκες σας είναι πολύ βασικές (μόνο προβολή υπογραφών, χωρίς τροποποίηση)  
  - Εργάζεστε μόνο με αρχεία εικόνας (χρησιμοποιήστε βιβλιοθήκη σάρωσης barcode)  
  - Ο προϋπολογισμός είναι εξαιρετικά περιορισμένος και η χειροκίνητη επεξεργασία είναι αποδεκτή  

## Πρακτικές Εφαρμογές

Ας δούμε πραγματικά σενάρια όπου αυτό έχει σημασία:

### 1. Σύστημα Διαχείρισης Συμβάσεων  
**Σενάριο:** Δημιουργείτε σύστημα που επικυρώνει υπογεγραμμένες συμβάσεις πριν τις αρχειοθετήσει.  

**Πώς βοηθά:** Αναζητά αυτόματα υπογραφές barcode που περιέχουν IDs συμβάσεων, επαληθεύει ότι ταιριάζουν με τη βάση δεδομένων σας και απορρίπτει έγγραφα με ελλιπείς ή μη έγκυρες υπογραφές. Αυτό αποτρέπει προβλήματα πριν τα έγγραφα εισέλθουν στην μόνιμη αρχειοθέτηση.

### 2. Αυτοματοποίηση Επεξεργασίας Τιμολογίων  
**Σενάριο:** Η εταιρεία σας λαμβάνει χιλιάδες τιμολόγια μηνιαίως, καθένα με υπογραφή barcode για επαλήθευση.  

**Πώς βοηθά:** Σαρώνει τα εισερχόμενα τιμολόγια για υπογραφές barcode, εξάγει πληροφορίες προμηθευτή και αριθμούς τιμολογίων, και κατευθύνει τα έγγραφα στη σωστή ροή έγκρισης. Εξαλείφει την ανάγκη χειροκίνητης ταξινόμησης και εισαγωγής δεδομένων.

### 3. Ροή Εργασίας Αναθεώρησης Εγγράφων  
**Σενάριο:** Νομικά έγγραφα χρειάζονται περιοδικές ενημερώσεις, απαιτώντας την αφαίρεση παλαιών υπογραφών πριν την επαναυπογραφή.  

**Πώς βοηθά:** Αφαιρεί προγραμματισμένα τις παλαιές υπογραφές barcode από έγγραφα που χρειάζονται αναθεώρηση, εξασφαλίζοντας καθαρά έγγραφα για τη νέα διαδικασία υπογραφής. Αποτρέπει σύγχυση σχετικά με το ποιες υπογραφές είναι τρέχουσες.

### 4. Συμμόρφωση και Έλεγχος  
**Σενάριο:** Η οργάνωσή σας πρέπει να επαληθεύσει ότι όλα τα έγγραφα σε ένα αρχείο έχουν έγκυρες υπογραφές.  

**Πώς βοηθά:** Επεξεργάζεται μαζικά το αρχείο, αναζητώντας σε κάθε αρχείο υπογραφές barcode και καταγράφει ποια έγγραφα λείπουν υπογραφές. Δημιουργεί αυτόματα εκθέσεις ελέγχου αντί για χειροκίνητη ανασκόπηση.

## Σκέψεις για Απόδοση

Όταν εργάζεστε με λειτουργίες υπογραφών σε παραγωγή, λάβετε υπόψη τους εξής παράγοντες απόδοσης:

### Διαχείριση Μνήμης  
Τα μεγάλα έγγραφα μπορούν να καταναλώσουν σημαντική μνήμη. Αν επεξεργάζεστε πολλά αρχεία, σκεφτείτε:  

- Επεξεργασία διαδοχικά αντί για φόρτωση όλων ταυτόχρονα  
- Χρήση σελιδοποίησης στις επιλογές αναζήτησης για επεξεργασία μεγάλων εγγράφων σε τμήματα  
- Έκκληση `signature.dispose()` (ή χρήση try‑with‑resources) για άμεση απελευθέρωση μνήμης  

### Βελτιστοποίηση Μαζικής Επεξεργασίας  
Επεξεργάζεστε πολλά έγγραφα; Αυτές οι στρατηγικές βοηθούν:  

- Επαναχρησιμοποίηση αντικειμένων ρυθμίσεων (π.χ. `BarcodeSearchOptions`) μεταξύ λειτουργιών  
- Παράλληλη επεξεργασία με `ExecutorService` της Java (προσοχή στη μνήμη)  
- Κρυφή αποθήκευση αποτελεσμάτων αναζήτησης αν χρειάζεται να κάνετε πολλαπλές ενέργειες στις ίδιες υπογραφές  

### Αποδοτικότητα Αρχείων I/O  
Οι λειτουργίες αρχείων μπορούν να γίνουν bottleneck:  

- Διαβάζετε από γρήγορο αποθηκευτικό (SSD αντί για δικτυακό δίσκο) όταν είναι δυνατόν  
- Αν διαγράφετε πολλές υπογραφές, κάντε το σε μία ενέργεια αντί για πολλαπλά αρχεία εξόδου  
- Κρατήστε συχνά προσπελαζόμενα έγγραφα στη μνήμη αν το σενάριό σας το επιτρέπει  

**Πρακτική συμβουλή:** Σε ένα έργο που συμμετείχα, μειώσαμε τον χρόνο επεξεργασίας κατά **60 %** απλώς με ομαδοποίηση λειτουργιών και επαναχρησιμοποίηση ρυθμίσεων αντί για δημιουργία νέων για κάθε έγγραφο.
