# .NET के लिए GroupDocs.Signature का उपयोग करके कस्टम क्रमांकन के साथ PDF मेटाडेटा हस्ताक्षर को कार्यान्वित करना

## परिचय

आज की डिजिटल दुनिया में, दस्तावेज़ों की प्रामाणिकता और अखंडता सुनिश्चित करना सर्वोपरि है। चाहे आप अनुबंध प्रबंधन प्रणालियों पर काम करने वाले डेवलपर हों या संवेदनशील जानकारी संभालने वाला संगठन, दस्तावेज़ों पर विश्वसनीय हस्ताक्षर करना बेहद ज़रूरी है। यह मार्गदर्शिका आपको कस्टम क्रमांकन का उपयोग करके PDF मेटाडेटा हस्ताक्षर को लागू करने में मदद करेगी। **.NET के लिए GroupDocs.Signature**—.NET अनुप्रयोगों में डिजिटल हस्ताक्षरों को सरल बनाने के लिए डिज़ाइन की गई एक शक्तिशाली लाइब्रेरी।

यह ट्यूटोरियल मेटाडेटा सिग्नेचर के लिए कस्टम सीरियलाइज़ेशन फ़ॉर्मेट बनाने और लागू करने पर केंद्रित है। यह एक ऐसी सुविधा है जो दस्तावेज़ों में एम्बेड किए जाने पर डेटा के प्रदर्शन के लचीलेपन को बढ़ाती है। .NET के लिए GroupDocs.Signature का लाभ उठाकर, आप इस बात पर नियंत्रण प्राप्त कर सकते हैं कि हस्ताक्षर-संबंधी डेटा, जैसे कि आईडी, लेखकत्व, दिनांक और अन्य मीट्रिक, आपकी PDF फ़ाइलों में कैसे सीरियलाइज़ और संग्रहीत किए जाते हैं।

**आप क्या सीखेंगे:**
- अपने वातावरण में .NET के लिए GroupDocs.Signature कैसे सेट अप और कॉन्फ़िगर करें
- अद्वितीय मेटाडेटा प्रारूपों को परिभाषित करने के लिए विशेषताओं का उपयोग करके कस्टम क्रमांकन को कार्यान्वित करना
- अनुकूलित मेटाडेटा हस्ताक्षरों के साथ दस्तावेज़ पर हस्ताक्षर करना
- डिजिटल हस्ताक्षरों के साथ कार्य करते समय प्रदर्शन को अनुकूलित करने के सर्वोत्तम अभ्यास

तकनीकी विवरण में जाने से पहले, आइए सुनिश्चित करें कि आपके पास सब कुछ तैयार है।

## .NET के लिए GroupDocs.Signature सेट अप करना

आरंभ करने के लिए, आपको स्थापित करने की आवश्यकता है **ग्रुपडॉक्स.हस्ताक्षर** लाइब्रेरी को अपने प्रोजेक्ट में जोड़ें। विभिन्न पैकेज मैनेजरों का उपयोग करके आप ऐसा कैसे कर सकते हैं, यहाँ बताया गया है:

### .NET सीएलआई:
```
dotnet add package GroupDocs.Signature
```

### पैकेज प्रबंधक:
```
Install-Package GroupDocs.Signature
```

### NuGet पैकेज प्रबंधक UI:
"GroupDocs.Signature" खोजें और अपने IDE से सीधे नवीनतम संस्करण स्थापित करें।

#### लाइसेंस प्राप्ति चरण:
- **मुफ्त परीक्षण**: सुविधाओं का पता लगाने के लिए निःशुल्क परीक्षण से शुरुआत करें।
- **अस्थायी लाइसेंस**: बिना किसी सीमा के विस्तारित परीक्षण के लिए अस्थायी लाइसेंस का अनुरोध करें।
- **खरीदना**यदि आपको उत्पादन उपयोग के लिए पूर्ण पहुंच की आवश्यकता है तो खरीदने पर विचार करें।

एक बार इंस्टॉल हो जाने पर, अपने प्रोजेक्ट में GroupDocs.Signature को निम्न प्रकार से आरंभ करें:

```csharp
using GroupDocs.Signature;

// इनपुट फ़ाइल पथ के साथ हस्ताक्षर वर्ग को आरंभ करें

```

## कार्यान्वयन मार्गदर्शिका

यह अनुभाग आपको कस्टम क्रमांकन तंत्र बनाने और दस्तावेजों पर हस्ताक्षर करने के लिए इसे लागू करने में मार्गदर्शन करेगा।

### मेटाडेटा हस्ताक्षरों के लिए कस्टम क्रमांकन बनाना

#### अवलोकन:
कस्टम क्रमांकन आपको यह निर्धारित करने की अनुमति देता है कि दस्तावेज़ों में मेटाडेटा एम्बेड करते समय विशिष्ट फ़ील्ड कैसे क्रमांकित किए जाएँ। यह विशेष रूप से विभिन्न प्रणालियों में डेटा की एकरूपता और पठनीयता सुनिश्चित करने के लिए उपयोगी है जो बाद में हस्ताक्षरित दस्तावेज़ का उपयोग कर सकते हैं।

#### चरण-दर-चरण कार्यान्वयन:

##### एक कस्टम डेटा हस्ताक्षर वर्ग परिभाषित करें
एक ऐसा वर्ग बनाएं जो आपके हस्ताक्षर डेटा का प्रतिनिधित्व करता हो तथा जिसमें क्रमांकन व्यवहार को नियंत्रित करने वाली विशेषताएं हों।

```csharp
using System;
using GroupDocs.Signature.Domain.Extensions;

class DocumentSignatureData
{
    [CustomSerialization]
    public class SignatureData
    {
        // SignID फ़ील्ड के लिए कस्टम प्रारूप का उपयोग करें
        [Format("SignID")]
        public string ID { get; set; }

        // लेखक फ़ील्ड के लिए कस्टम प्रारूप
        [Format("SAuth")]
        public string Author { get; set; }

        // दिनांक प्रारूप को किसी विशिष्ट पैटर्न के साथ अनुकूलित करें
        [Format("SDate", "yyyy-MM-dd")]
        public DateTime Signed { get; set; }

        // संख्या को दो दशमलव स्थानों के साथ प्रारूपित करें
        [Format("SDFact", "N2")]
        public decimal DataFactor { get; set; }

        // इस फ़ील्ड को क्रमांकन से बाहर रखें
        [SkipSerialization]
        public string Comments { get; set; }
    }
}
```

##### स्पष्टीकरण:
- **[कस्टम सीरियलाइजेशन]**: संपूर्ण वर्ग को कस्टम क्रमांकन के लिए चिह्नित करता है।
- **[Format("FieldName", "Patern")])**: यह निर्दिष्ट करता है कि किसी विशेष गुण को किस प्रकार क्रमबद्ध किया जाना चाहिए, जिसमें उसकी कुंजी और स्वरूपण पैटर्न भी शामिल है।
- **[SkipSerialization]**: गुणों को क्रमबद्ध होने से बाहर रखता है।

### मेटाडेटा और कस्टम क्रमांकन के साथ दस्तावेज़ पर हस्ताक्षर करना

#### अवलोकन:
इस अनुभाग में, आप दस्तावेज़ पर हस्ताक्षर करने के लिए कस्टम क्रमांकन वर्ग का उपयोग करेंगे। इसमें मेटाडेटा हस्ताक्षर सेट करना और .NET के लिए GroupDocs.Signature का उपयोग करके उन्हें लागू करना शामिल है।

##### क्रमशः:

###### एन्क्रिप्शन सेटअप करें
अपने हस्ताक्षर मेटाडेटा को सुरक्षित करने के लिए डेटा एन्क्रिप्शन लागू करें।

```csharp
using System.IO;
using GroupDocs.Signature.Domain;

// एक एन्क्रिप्शन ऑब्जेक्ट बनाएँ (उदाहरण के लिए, CustomXOREncryption)
IDataEncryption encryption = new CustomXOREncryption();
```

###### मेटाडेटा साइन विकल्प कॉन्फ़िगर करें
कस्टम क्रमांकन और एन्क्रिप्शन सहित हस्ताक्षर के लिए विकल्प सेट करें।

```csharp
using GroupDocs.Signature.Options;
using GroupDocs.Signature.Domain;

MetadataSignOptions options = new MetadataSignOptions()
{
    DataEncryption = encryption
};
```

###### कस्टम हस्ताक्षर डेटा ऑब्जेक्ट बनाएँ
विशिष्ट हस्ताक्षर विवरण के साथ अपने कस्टम डेटा वर्ग को इंस्टैंसिएट करें।

```csharp
documentSignatureData = new DocumentSignatureData.SignatureData
{
    ID = Guid.NewGuid().ToString(),
    Author = Environment.UserName,
    Signed = DateTime.Now,
    DataFactor = 11.22M
};
```

###### हस्ताक्षर मेटाडेटा जोड़ें
विकल्पों में विभिन्न मेटाडेटा फ़ील्ड जोड़ें.

```csharp
using GroupDocs.Signature.Domain;

WordProcessingMetadataSignature mdSignature = new WordProcessingMetadataSignature("Signature", documentSignatureData);
WordProcessingMetadataSignature mdAuthor = new WordProcessingMetadataSignature("Author", "Mr.Scherlock Holmes");
WordProcessingMetadataSignature mdDocId = new WordProcessingMetadataSignature("DocumentId", Guid.NewGuid().ToString());

options.Add(mdSignature).Add(mdAuthor).Add(mdDocId);
```

###### दस्तावेज़ पर हस्ताक्षर करें
अपने दस्तावेज़ पर हस्ताक्षर करने के लिए कॉन्फ़िगर किए गए विकल्प लागू करें.

```csharp
using GroupDocs.Signature;

string filePath = "YOUR_DOCUMENT_DIRECTORY";
string outputFilePath = Path.Combine("YOUR_OUTPUT_DIRECTORY", "SignedDocument.pdf");

using (Signature signature = new Signature(filePath))
{
    // दस्तावेज़ पर हस्ताक्षर करें और उसे सहेजें
    SignResult signResult = signature.Sign(outputFilePath, options);
}
```

### समस्या निवारण युक्तियों:
- सुनिश्चित करें कि आपके फ़ाइल पथ सही ढंग से निर्दिष्ट हैं.
- सत्यापित करें कि कस्टम क्रमांकन के लिए सभी आवश्यक विशेषताएँ ठीक से सेट की गई हैं।
- जांचें कि GroupDocs.Signature लाइब्रेरी कस्टम सुविधाओं का समर्थन करने के लिए अद्यतन की गई है।

## व्यावहारिक अनुप्रयोगों

मेटाडेटा हस्ताक्षरों को अनुकूलित करने की क्षमता के कई वास्तविक-विश्व अनुप्रयोग हैं:

1. **अनुबंध प्रबंधन**: दस्तावेज़ों में मानकीकृत प्रारूप में अनुबंध आईडी और हस्ताक्षर तिथियों को एम्बेड करने के लिए कस्टम प्रारूपों का उपयोग करें।
2. **दस्तावेज़ संस्करण नियंत्रण**: संस्करण संख्या और लेखकत्व विवरण को सीधे मेटाडेटा में संलग्न करें, जिससे पता लगाने की क्षमता सुनिश्चित हो सके।
3. **ई-कॉमर्स लेनदेन**: लेनदेन आईडी और राशि को पीडीएफ चालान या रसीदों में सुरक्षित रूप से एम्बेड करें।
4. **कानूनी दस्तावेज़ीकरण**: ऑडिट के दौरान आसानी से प्राप्त करने के लिए पूर्वनिर्धारित प्रारूप में केस संख्या और कानूनी शर्तें जोड़ें।

सीआरएम या ईआरपी प्लेटफॉर्म जैसे अन्य प्रणालियों के साथ एकीकरण मेटाडेटा निष्कर्षण और प्रसंस्करण को स्वचालित करके दस्तावेज़ प्रबंधन वर्कफ़्लो को और बढ़ा सकता है।

## प्रदर्शन संबंधी विचार

डिजिटल हस्ताक्षरों के साथ काम करते समय, प्रदर्शन को अनुकूलित करना महत्वपूर्ण है:

- **अतुल्यकालिक प्रसंस्करण**: अवरुद्ध संचालन से बचने के लिए अतुल्यकालिक विधियों का उपयोग करें।
- **संसाधन प्रबंधन**मेमोरी लीक या अत्यधिक CPU उपयोग को रोकने के लिए संसाधनों का उचित प्रबंधन करें।
- **प्रचय संसाधन**एकाधिक दस्तावेज़ों को संभालते समय, दक्षता में सुधार के लिए बैच प्रोसेसिंग तकनीकों पर विचार करें।

इन दिशानिर्देशों का पालन करके और .NET के लिए GroupDocs.Signature की सुविधाओं का उपयोग करके, आप अपने अनुप्रयोगों में मजबूत मेटाडेटा हस्ताक्षर समाधान को प्रभावी ढंग से लागू कर सकते हैं।
