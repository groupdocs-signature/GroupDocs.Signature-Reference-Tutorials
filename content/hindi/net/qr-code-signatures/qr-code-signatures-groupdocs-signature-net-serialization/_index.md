# GroupDocs.Signature का उपयोग करके .NET में कस्टम क्रमांकन के साथ QR कोड हस्ताक्षर लागू करें

## परिचय

आज के डिजिटल युग में, दस्तावेज़ प्रामाणिकता का प्रबंधन कानून, व्यवसाय और सॉफ़्टवेयर विकास जैसे विभिन्न क्षेत्रों में अत्यंत महत्वपूर्ण है। GroupDocs.Signature for .NET आपके अनुप्रयोगों में कस्टम डेटा क्रमांकन के साथ QR कोड हस्ताक्षरों को सहजता से एकीकृत करने की शक्तिशाली क्षमताएँ प्रदान करता है।

यह ट्यूटोरियल .NET के लिए GroupDocs.Signature में कस्टम क्रमांकन का उपयोग करके QR कोड हस्ताक्षरों को लागू करने, दस्तावेज़ सुरक्षा को बढ़ाने और हस्ताक्षर डेटा को संभालने के लिए एक अनुकूलन योग्य दृष्टिकोण प्रदान करने का पता लगाता है।

**आप क्या सीखेंगे:**
- क्यूआर कोड में कस्टम डेटा क्रमांकन की मूल बातें
- .NET के लिए GroupDocs.Signature के लिए पर्यावरण सेटअप
- कस्टम विकल्पों के साथ QR कोड हस्ताक्षरों को कार्यान्वित करना और खोजना
- वास्तविक दुनिया के परिदृश्यों में व्यावहारिक अनुप्रयोग

कार्यान्वयन में उतरने से पहले, आइए कुछ पूर्वापेक्षाओं की समीक्षा करें।

## कार्यान्वयन मार्गदर्शिका

इस अनुभाग में, हम कवर करेंगे कि .NET के लिए GroupDocs.Signature का उपयोग करके QR-कोड हस्ताक्षर और खोज के लिए कस्टम डेटा क्रमांकन कैसे लागू किया जाए।

### क्यूआर-कोड हस्ताक्षरों के लिए कस्टम डेटा क्रमांकन

**अवलोकन:**
कस्टम डेटा क्रमांकन आपको अपने हस्ताक्षर डेटा के लिए विशिष्ट प्रारूप परिभाषित करने की अनुमति देता है, जो आपके अनुप्रयोग की आवश्यकताओं के अनुसार सूचना संरचना के लिए आवश्यक है।

#### चरण 1: हस्ताक्षर डेटा वर्ग को परिभाषित करें

हस्ताक्षर डेटा रखने वाला एक वर्ग बनाएं:

```csharp
using System;
using GroupDocs.Signature.Domain;

[CustomSerialization]
private class DocumentSignatureData
{
    [Format("SignID")]
    public string ID { get; set; }

    [Format("SAuth")]
    public string Author { get; set; }

    [Format("SDate", "yyyy-MM-dd")]
    public DateTime Signed { get; set; }

    [Format("SDFact", "N2")]
    public decimal DataFactor { get; set; }

    // टिप्पणियाँ फ़ील्ड को क्रमांकन से बाहर रखें
    [SkipSerialization]
    public string Comments { get; set; }
}
```

**स्पष्टीकरण:**
- **कस्टमसीरियलाइजेशन:** इस वर्ग को कस्टम डेटा हैंडलिंग के लिए चिह्नित करता है।
- **प्रारूप विशेषता:** प्रारूप प्रकार सहित प्रत्येक गुण को क्रमबद्ध कैसे किया जाना चाहिए, यह परिभाषित करता है।
- **सीरियलाइजेशन छोड़ें:** क्रमांकन से कुछ गुणों को बाहर रखा गया है।

#### चरण 2: कस्टम विकल्पों के साथ QR-कोड हस्ताक्षरों की खोज करना

**अवलोकन:**
आप कस्टम विकल्पों का उपयोग करके QR-कोड हस्ताक्षरों के लिए दस्तावेज़ खोज सकते हैं, जिससे कुशल दस्तावेज़ सत्यापन सुनिश्चित होता है।

##### खोज सेट अप करना

```csharp
using System;
using System.Collections.Generic;
using GroupDocs.Signature;
using GroupDocs.Signature.Options;
using GroupDocs.Signature.Domain.Extensions;

public class SearchForQRCodeWithCustomOptions
{
    public static void Run()
    {
        string filePath = "YOUR_DOCUMENT_DIRECTORY";

        using (Signature signature = new Signature(filePath))
        {
            IDataEncryption encryption = new CustomXOREncryption();

            QrCodeSearchOptions options = new QrCodeSearchOptions()
            {
                AllPages = true,
                DataEncryption = encryption
            };

            try
            {
                List<QrCodeSignature> signatures = signature.Search<QrCodeSignature>(options);

                foreach (var qrCodeSignature in signatures)
                {
                    DocumentSignatureData documentSignatureData = qrCodeSignature.GetData<DocumentSignatureData>();
                    if (documentSignatureData != null)
                    {
                        Console.WriteLine("QRCode signature found with details:");
                        Console.WriteLine("ID: {0}, Author: {1}, Signed: {2}, DataFactor: {3}", 
                            documentSignatureData.ID, documentSignatureData.Author,
                            documentSignatureData.Signed.ToShortDateString(), documentSignatureData.DataFactor);
                    }
                }
            }
            catch (Exception ex)
            {
                Console.WriteLine("Error during search process: " + ex.Message);
            }
        }
    }
}
```

**स्पष्टीकरण:**
- **कस्टमXORE एन्क्रिप्शन:** अतिरिक्त सुरक्षा के लिए कस्टम डेटा एन्क्रिप्शन लागू करता है।
- **QrCodeSearchविकल्प:** कस्टम डेटा एन्क्रिप्शन लागू करने सहित QR कोड खोज सेटिंग्स कॉन्फ़िगर करता है।
- **GetData विधि:** किसी प्राप्त हस्ताक्षर से क्रमबद्ध डेटा निकालता है।

### समस्या निवारण युक्तियों

- फ़ाइल-नहीं-मिले अपवादों से बचने के लिए सुनिश्चित करें कि दस्तावेज़ पथ सही ढंग से निर्दिष्ट किया गया है।
- रनटाइम त्रुटियों को रोकने के लिए सत्यापित करें कि सभी निर्भरताएं स्थापित और अद्यतित हैं।

## व्यावहारिक अनुप्रयोगों

क्रमांकन के साथ कस्टम क्यूआर कोड हस्ताक्षर विभिन्न परिदृश्यों में लागू किए जा सकते हैं:

1. **कानूनी अनुबंध:** कानूनी दस्तावेजों में अद्वितीय, एन्क्रिप्टेड हस्ताक्षर एम्बेड करके अनुबंध सुरक्षा को बढ़ाएं।
2. **वित्तीय दस्तावेज:** सुरक्षित हस्ताक्षर सत्यापन के माध्यम से वित्तीय विवरणों की प्रामाणिकता सुनिश्चित करें।
3. **पहचान सत्यापन:** क्रमबद्ध क्यूआर कोड डेटा का उपयोग करके पहचान सत्यापित करने के लिए एक मजबूत प्रणाली लागू करें।
4. **आपूर्ति श्रृंखला प्रबंधन:** कस्टम क्रमांकित क्यूआर कोड के साथ शिपमेंट दस्तावेज़ को ट्रैक और प्रमाणित करें।
5. **स्वास्थ्य देखभाल रिकॉर्ड:** एन्क्रिप्टेड क्यूआर हस्ताक्षरों को एकीकृत करके रोगी रिकॉर्ड सुरक्षित करें।
