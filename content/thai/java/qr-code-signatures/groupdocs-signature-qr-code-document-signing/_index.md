# การลงนามเอกสารหลักด้วยรหัส QR โดยใช้ GroupDocs.Signature สำหรับ Java

## การแนะนำ

ในยุคดิจิทัล การจัดการและรักษาความปลอดภัยข้อมูลยาอย่างมีประสิทธิภาพเป็นสิ่งสำคัญอย่างยิ่งต่อการปฏิบัติตามกฎระเบียบและประสิทธิภาพการดำเนินงาน การผสานรวมข้อมูลผลิตภัณฑ์ที่ครอบคลุมลงในเอกสารอาจเป็นเรื่องท้าทาย บทช่วยสอนนี้จะสาธิตวิธีการใช้ **GroupDocs.Signature สำหรับ Java** เพื่อเข้ารหัสข้อมูลบาร์โค้ดอุตสาหกรรมสุขภาพ (HIBC) ภายในรหัส QR และลงนามเอกสารได้อย่างราบรื่น

### สิ่งที่คุณจะได้เรียนรู้:
- ตั้งค่า GroupDocs.Signature สำหรับ Java
- สร้างอินสแตนซ์ของ HIBCLICPrimaryData, HIBCLICSecondaryAdditionalData และแบบฟอร์มรวมกัน
- ลงนามเอกสารโดยใช้รหัส QR ที่เข้ารหัสข้อมูลผลิตภัณฑ์โดยละเอียด
- เพิ่มประสิทธิภาพการทำงานพร้อมทั้งบริหารจัดการทรัพยากรอย่างมีประสิทธิผล

### ข้อกำหนดการตั้งค่าสภาพแวดล้อม
ตรวจสอบให้แน่ใจว่าสภาพแวดล้อมการพัฒนาของคุณได้รับการกำหนดค่าให้ใช้ Maven หรือ Gradle ซึ่งจะช่วยลดความซับซ้อนในการจัดการการอ้างอิงและการสร้างโครงการ

## การตั้งค่า GroupDocs.Signature สำหรับ Java

### ข้อมูลการติดตั้ง

**เมเวน**
```xml
<dependency>
    <groupId>com.groupdocs</groupId>
    <artifactId>groupdocs-signature</artifactId>
    <version>23.12</version>
</dependency>
```

**แกรเดิล**
```gradle
implementation 'com.groupdocs:groupdocs-signature:23.12'
```

**ดาวน์โหลดโดยตรง**: ดาวน์โหลดเวอร์ชันล่าสุดได้จาก [GroupDocs.Signature สำหรับรุ่น Java](https://releases-groupdocs.com/signature/java/).

### ขั้นตอนการขอใบอนุญาต
1. **ทดลองใช้ฟรี**:เริ่มต้นด้วยการดาวน์โหลดรุ่นทดลองใช้เพื่อทดสอบฟังก์ชันพื้นฐาน
2. **ใบอนุญาตชั่วคราว**:รับสิทธิ์เข้าถึงเต็มรูปแบบโดยไม่มีข้อจำกัดในช่วงระยะเวลาประเมินผลของคุณ
3. **ซื้อ**:ควรพิจารณาซื้อใบอนุญาตสำหรับโครงการระยะยาว

#### การเริ่มต้นและการตั้งค่าขั้นพื้นฐาน
เมื่อติดตั้งแล้วให้เริ่มต้นการทำงาน `Signature` วัตถุที่มีเส้นทางไฟล์ของเอกสารที่คุณต้องการลงนาม:
```java
String filePath = "Sample.pdf";
Signature signature = new Signature(filePath);
```

## คู่มือการใช้งาน

### สร้างข้อมูลหลักของ HIBC LIC
**ภาพรวม**:ส่วนนี้สาธิตวิธีการสร้างและกำหนดค่าอินสแตนซ์ของ `HIBCLICPrimaryData`ซึ่งมีข้อมูลผลิตภัณฑ์ที่สำคัญ

#### ขั้นตอนที่ 1: เริ่มต้นวัตถุข้อมูลหลัก
```java
HIBCLICPrimaryData primaryData = new HIBCLICPrimaryData();
```

#### ขั้นตอนที่ 2: ตั้งค่าคุณสมบัติที่จำเป็น
- **หมายเลขผลิตภัณฑ์หรือแคตตาล็อก**:ตัวระบุเฉพาะสำหรับผลิตภัณฑ์
- **รหัสประจำตัวผู้ติดฉลาก**: ระบุผู้ผลิต
- **รหัสหน่วยวัด**: ระบุหน่วยการวัด

```java
primaryData.setProductOrCatalogNumber("12345");
primaryData.setLabelerIdentificationCode("A999");
primaryData.setUnitOfMeasureID(1);
```

### สร้างข้อมูลเพิ่มเติมรองของ HIBC LIC
**ภาพรวม**:ส่วนนี้จะครอบคลุมถึงการสร้างและการกำหนดค่าอินสแตนซ์ของ `HIBCLICSecondaryAdditionalData`ซึ่งรวมถึงรายละเอียดเพิ่มเติม เช่น วันหมดอายุ และหมายเลขล็อต

#### ขั้นตอนที่ 1: เริ่มต้นวัตถุข้อมูลรอง
```java
HIBCLICSecondaryAdditionalData secondaryData = new HIBCLICSecondaryAdditionalData();
```

#### ขั้นตอนที่ 2: ตั้งค่าคุณสมบัติเพิ่มเติม
- **วันหมดอายุ**:ใช้วันที่ปัจจุบันเพื่อการสาธิต
- **จำนวน, หมายเลขล็อต, หมายเลขซีเรียล**: กำหนดข้อมูลจำเพาะของผลิตภัณฑ์
- **วันที่ผลิตและลักษณะการเชื่อมต่อ**: กำหนดรายละเอียดการผลิต

```java
secondaryData.setExpiryDate(new Date());
secondaryData.setExpiryDateFormat(HIBCLICDateFormat.MMDDYY);
secondaryData.setQuantity(30);
secondaryData.setLotNumber("LOT123");
secondaryData.setSerialNumber("SERIAL123");
secondaryData.setDateOfManufacture(new Date());
secondaryData.setLinkCharacter('S');
```

### รวมข้อมูลหลักและข้อมูลรองของ HIBC LIC
**ภาพรวม**:เรียนรู้วิธีการผสานข้อมูลหลักและข้อมูลรองให้เป็นข้อมูลเดียว `HIBCLICCombinedData` วัตถุสำหรับการประมวลผลแบบคล่องตัว

#### ขั้นตอนที่ 1: เริ่มต้นวัตถุข้อมูลรวม
```java
HIBCLICCombinedData combinedData = new HIBCLICCombinedData();
```

#### ขั้นตอนที่ 2: ตั้งค่าข้อมูลหลักและข้อมูลรอง
- เชื่อมโยงชุดข้อมูลทั้งสองเพื่อสร้างโครงสร้างข้อมูลที่สมบูรณ์

```java
combinedData.setPrimaryData(primaryData);
combinedData.setSecondaryAdditionalData(secondaryData);
```

### ลงนามในเอกสารที่มีรหัส QR ที่มีข้อมูลรวมของ HIBC LIC
**ภาพรวม**:ส่วนสุดท้ายนี้สาธิตวิธีการลงนามเอกสารโดยใช้รหัส QR ที่เข้ารหัสข้อมูล HIBC รวม

#### ขั้นตอนที่ 1: กำหนดเส้นทางไฟล์
```java
String filePath = "YOUR_DOCUMENT_DIRECTORY/Sample.pdf";
String outputFilePath = "YOUR_OUTPUT_DIRECTORY/SignWithQRCodeHIBCLICCombinedData/" + fileName;
```

#### ขั้นตอนที่ 2: ตั้งค่าตัวเลือกการลงนามรหัส QR
- **ประเภทการเข้ารหัส**: ใช้ `QrCodeTypes.HIBCLICQR` เพื่อระบุประเภทการเข้ารหัส
- **การกำหนดข้อมูล**:ส่งข้อมูลรวมเพื่อรวมไว้ในรหัส QR

```java
Signature signature = new Signature(filePath);
try {
    QrCodeSignOptions options = new QrCodeSignOptions();
    options.setEncodeType(QrCodeTypes.HIBCLICQR);
    options.setData(combinedData);

    // ลงนามและบันทึกเอกสาร
    signature.sign(outputFilePath, options);
} finally {
    if (signature != null) signature.dispose();
}
```

## การประยุกต์ใช้งานจริง
1. **การปฏิบัติตามข้อกำหนดด้านเภสัชกรรม**:ปรับปรุงการปฏิบัติตามมาตรฐานการกำกับดูแลให้มีประสิทธิภาพยิ่งขึ้นโดยใช้การผสานรวมนี้
2. **การจัดการห่วงโซ่อุปทาน**:เพิ่มความสามารถในการตรวจสอบย้อนกลับของผลิตภัณฑ์ยาผ่าน QR code ในเอกสาร
3. **การบูรณาการระบบการดูแลสุขภาพ**:ฝังข้อมูลผลิตภัณฑ์ที่ครอบคลุมลงในบันทึกการดูแลสุขภาพเพื่อความปลอดภัยของผู้ป่วยที่ดีขึ้น

## การพิจารณาประสิทธิภาพ
- **เพิ่มประสิทธิภาพการใช้ทรัพยากร**:รับรองการจัดการหน่วยความจำที่มีประสิทธิภาพโดยการกำจัด `Signature` วัตถุหลังการดำเนินการ
- **แนวทางปฏิบัติที่ดีที่สุด**:อัปเดตเป็น GroupDocs.Signature เวอร์ชันล่าสุดเป็นประจำเพื่อปรับปรุงประสิทธิภาพและแก้ไขจุดบกพร่อง

เมื่อทำตามคำแนะนำนี้ คุณจะได้เรียนรู้วิธีการสร้างออบเจ็กต์ข้อมูลหลักและรองของ HIBC LIC รวมข้อมูลเหล่านั้นให้เป็นเอนทิตีเดียว และลงนามในเอกสารด้วยรหัส QR โดยใช้ GroupDocs.Signature สำหรับ Java ทักษะเหล่านี้ช่วยเพิ่มความปลอดภัยของเอกสารและรับรองการปฏิบัติตามข้อกำหนดในอุตสาหกรรมยา

## ส่วนคำถามที่พบบ่อย
1. **ข้อมูล HIBC คืออะไร?**
   - ข้อมูลบาร์โค้ดอุตสาหกรรมสุขภาพ (HIBC) ประกอบด้วยข้อมูลผลิตภัณฑ์ที่จำเป็นที่ใช้ในอุตสาหกรรมการดูแลสุขภาพและเภสัชกรรม
2. **ฉันสามารถใช้ GroupDocs.Signature สำหรับบาร์โค้ดประเภทอื่นได้หรือไม่**
   - ใช่ GroupDocs.Signature รองรับรูปแบบบาร์โค้ดที่หลากหลายนอกเหนือจากรหัส QR
3. **จะเกิดอะไรขึ้นหากรูปแบบเอกสารของฉันไม่ใช่ PDF?**
   - GroupDocs.Signature รองรับรูปแบบเอกสารหลายรูปแบบ รวมถึง Word และ Excel
4. **ฉันจะจัดการข้อยกเว้นระหว่างการลงนามได้อย่างไร**
   - นำบล็อก try-catch มาใช้เพื่อจัดการข้อยกเว้นอย่างมีประสิทธิภาพและรับรองการล้างทรัพยากร
5. **จำนวนรหัส QR ต่อเอกสารมีการจำกัดหรือไม่?**
   - ไม่มีข้อจำกัดโดยธรรมชาติ แต่ควรพิจารณาถึงผลกระทบต่อประสิทธิภาพเมื่อเพิ่มโค้ดจำนวนมาก
